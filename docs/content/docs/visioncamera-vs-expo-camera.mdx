title: VisionCamera vs Expo Camera
description: Choosing between VisionCamera and Expo Camera based on documented API surface, platform support, and native integration requirements

This page compares VisionCamera and Expo Camera using current VisionCamera docs/code and the official [Expo Camera documentation](https://docs.expo.dev/versions/latest/sdk/camera/).
It is intentionally scoped to documented capabilities, not popularity, reliability, or benchmark claims.

### TL;DR

Use Expo Camera for Expo Go, Web, and the simplest possible camera screen.

Use VisionCamera for anything that needs deeper native camera control: pro capture controls, realtime frame access, ML, depth, HDR/RAW, multi-cam, custom outputs, or native plugins.

VisionCamera is easy to use through its declarative API, but it intentionally exposes much more of the native camera pipeline.

### Quick comparison

| Need | Pick | Why |
| --- | --- | --- |
| Expo Go | Expo Camera | It is included in Expo Go. |
| Web | Expo Camera | Expo Camera documents Web support. |
| Simplest camera screen | Expo Camera | `CameraView` is the smallest API surface when preview/photo/video/barcodes are enough. |
| Camera-first app | VisionCamera | More control, more outputs, and a deeper native API surface. |
| Realtime ML or frame access | VisionCamera | [Frame Processors](frame-output) stream GPU-backed frames to JS worklets and native plugins. |
| Pro capture features | VisionCamera | Manual controls, HDR/RAW, constraints, depth, multi-cam, and custom outputs. |

### Expo Camera gives you

- Expo Go support.
- Web support.
- Extremely simple setup and API when `CameraView` covers your use case.

### VisionCamera gives you

- Pro capture control: [manual AE/AF/AWB](locking-ae-af-awb), configurable `focusTo(...)`, [exposure bias](exposure-bias), [low light boost](low-light-boost), [custom FPS](fps), [stabilization](video-stabilization), [Photo HDR](photo-hdr), [Video HDR/Dynamic Ranges](video-hdr), [RAW/DNG photos](raw-photos), Android [Camera Extensions](camera-extensions), maximum-resolution photo paths, photo callbacks, output quality settings, codecs, and location tags.
- Realtime pipelines: [Frame Processors](frame-output), pixel/color access, GPU resizing for ML, [Skia interop](skia-frame-processors), [Depth frames](depth-output), MLKit barcode scanning, iOS native [Object Output](object-output), and native Frame Processor plugins.
- Native session control: multiple outputs, the [Constraints API](constraints), explicit lifecycle control, imperative `CameraSession`/`CameraController` APIs, specific/external camera selection, [multi-camera sessions](multi-camera), persistent recording while switching cameras, coordinate conversions, and custom native `CameraOutput`s.

### Expo projects

Both can be used in Expo projects, but the integration path is different.

Expo Camera is an Expo SDK package and is documented as included in Expo Go.
VisionCamera is a third-party native React Native package; in an Expo app, follow ["Getting Started"](/docs) and use `expo prebuild`, `expo run:*`, or a development build.

### Note on speed

VisionCamera is designed for high-performance native camera pipelines, but this page does not include independent Expo-Camera-vs-VisionCamera benchmarks.
